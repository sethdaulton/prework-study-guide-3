HTML

  • The Biggest heading "h1" should only appear once on each page
  • "Semantic HTML element" is the name of the element that describes its content and the role it plays on a web page
  • "Element" refers to the entire element from opening tag to closing tag
  • "Tag" refers only to what appears inside the angle brackets
  • "src" defines the location of the image
  • "Alt text" can improve search engine rankings for your website
  • "!DOCTYPE html" tells the browser what document type it should expect when it is opened
  • Declarations are not case sensitive; HTML elements are case sensitive
  • "Metadata" is the data about our webpage that the browser uses to effectively serve our webpage to users
  • "body" contains content that is shown to users

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • . card: the period preceding "card" designates this as a class selector. Any HTML element assigned to the class "card" will gain these property assignments from the CSS

Git

  • "git status": checks what branch we are currently on
  • "git checkout -b branch-name": creates a new branch and switches to it

Git Flow

  • git status
  • git add -A
  • git commit -m ":"add comment here"
  • git pull origin main
  • git push origin "feature/"name-of-branch"
  • Go to GitHub to manage new branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is simply a set of instructions that tells the computer how to perform a certain task
  • It is important to remember that since control flow dictates that JavaScript files are executed from top to bottom, order matters!
